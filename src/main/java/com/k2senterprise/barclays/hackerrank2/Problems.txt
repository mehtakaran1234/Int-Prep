You need to simulate an ATM that handles deposits and withdrawals using five denominations: 20, 50, 100, 200, and 500.Core LogicInput Format: 6 integers.If the first integer is -1, it's a Deposit. The following 5 integers represent the count of each denomination added to the machine.If the first integer is positive, it's a Withdrawal request for that specific amount.Withdrawal Rules:The machine is greedy: It always tries to use the highest denomination (500) first, then 200, and so on.Failure Condition: If the machine cannot fulfill the exact amount using its current stock and the greedy approach, the transaction fails.Atomicity: If a withdrawal fails, no notes are deducted from the machine.Output:Deposit: No output.Withdrawal: Return an array of 5 integers (count of each note handed out). If it fails, return -1.Key Edge Case MentionedThe text highlights a specific scenario: if a user asks for 600 and the machine has a 500 note and three 200 notes, it will try to use the 500 first. It will then be unable to fulfill the remaining 100 with 200s, and the request will be rejected. It will not "be smart" and use three 200s instead.Suggested Implementation StepsState Management: Maintain a global or class-level array/map to store the current count of the 5 denominations.Greedy Subtraction: For a withdrawal, iterate backwards through the denominations (500 down to 20).countToGive = min(needed / denom, available[denom])Validation: After the loop, if the remaining amount is 0, update the machine's stock and return the counts. Otherwise, discard the changes and return -1.
